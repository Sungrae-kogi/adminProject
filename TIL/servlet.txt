1.서블릿 핵심 객체

서블릿 프로그램의 생소함은 서블릿 객체가 *서블릿 컨테이너*에 의해 생성되고 실행되기 때문이다.
*서블릿 컨테이너*는 서블릿 객체를 생성하고 실행하는 과정에서 다양한 객체들을 생성하여 서블릿 객체에 전달한다.

*중요!
>서블릿 프로그램에서 잊지 말아야할 핵심개념은 서블릿 객체는 *서블릿 컨테이너*가 생성하고 관리한다는것이다.
서블릿 객체는 싱글톤으로 관리하여 객체를 딱 하나 생성해놓고 공유하여 사용한다. 즉, 최초의 요청이면 메모리 로딩 -> 객체생성 -> init() 호출  이후의 요청시에는 서블릿 객체가 생성되었기때문에 init()은 호출되지않고 service()만 호출시마다 호출된다.
그리고 서블릿 객체가 가진 콜백 메소드 역시 적절한 시점에 *서블릿 컨테이너*가 자동으로 호출해준다.

ex) 서블릿 객체가 생성된 직후에는 init() 메소드를 *서블릿 컨테이너가*가 자동으로 호출해주고 
스레드 풀에서 새로운 스레드 객체가 할당되면 해당 스레드가 service()메소드를 호출한다. 
이때 중요한점은 콜백 메소드가 호출되기 위해서는 콜백 메소드의 매개변수로 선언된 데이터가 반드시 인자로 전달되어야 한다는 것이다. 
그리고 이런 인자에 해당하는 객체에 대한 생성 역시 서블릿 컨테이너가 담당한다.


/* 중요개념!!!!!!!
Client -> HTTP 요청 -> Servlet Container가 HttpServletRequest 객체를 생성
	->HTTP 요청 프로토콜에 설정된 정보들을 추출하여 HttpServletRequest 객체에 설정한다.
	-> 이렇게 생성된 HttpServletRequest 객체는 service()나 doPost(), doGet()메소드 호출시 인자로 전달된다.
	 
HttpServletRequest 객체에는 HTTP 요청 프로토콜의 모든 정보가 설정된다. 따라서 HttpServletRequest를 HTTP 요청 프로토콜이라고 봐도 무방하다.
따라서 HttpServletRequest를 인자로 전달받은 서블릿에서는 HttpServlet을 통해 사용자가 입력한 정보뿐아니라 브라우저의 정보같은 다양한 정보의 추출 및 사용이 가능하다.
*/

HttpServletResponse 객체 역시 서블릿 객체의 service()메소드 매개변수로 선언되며, *서블릿 컨테이너*가 생성하여 전달한다.

웹 브라우저에서 사용자의 요청에 따라 서버가 실행할 수 있는 자바 프로그램은 서블릿뿐이다.
서블릿 API를 상속하면 웹에서 동작하는 기본 조건을 갖출 수 있다.
서블릿 구현시 서블릿 API를 상속받거나 필요한 메서드를 재정의해야 함

☆Servlet 인터페이스
서블릿 프로그램을 개발할 때 반드시 구현해야 하는 메소드를 선언하고 있는 인터페이스
ex) init(), destroy(), getServletConfig(), getServletInfo() 등

☆GenericServlet 클래스
Servlet 인터페이스를 상속하여 클라이언트-서버 환경에서 서버단의 애플리케이션으로 필요한 기능을 구현한 추상 클래스
service() 메소드를 제외한 모든 메소드를 재정의 함
GenericServlet 클래스를 상속하면 애플리케이션의 프로토콜에 따라 메소드 재정의 구문 적용 필요

☆HttpServlet 클래스
GenericServlet 클래스를 상속하여 service() 메소드를 재정의함으로써 HTTP 프로토콜에 알맞은 동작을 수행하도록 구현한 클래스
HTTP 프로토콜 기반으로 브라우저로부터 요청을 전달받아서 처리하도록 하는 클래스
service() 메소드에는 요청방식(GET 또는 POST)에 따라 doGet(), doPost() 등 정해진 사양의 메소드가 호출 되도록 구현
반드시 구현해야 하는 메소드는 없음


콜백 메소드 -> 어떤 객체에서 상황이 발생할때 컨테이너가 자동으로 호출하여 실행되는 메소드, 이러한 메소드들이 서블릿을 실행 함
ex)
init() : 클라이언트로부터 최초의 Servlet 실행 요청이 있을 때 한번 실행되는 초기화메소드
service() : 클라이언트로부터 요청이 있을 때마다 n번 실행되는 실제 서블릿이 처리하는 로직 메소드
destroy() : 서블릿 객체가 메모리에서 삭제될때 , 1번 자원을 해제하는 메소드

WEB-INF 디렉터리에 있는 web.xml 파일은 웹 애플리케이션에 관한 환경설정 파일로서 웹서버가 시작할 때 web.xml 파일에 설정된 내용대로 서비스를 올려 줌
