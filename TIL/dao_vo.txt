DAO - Data Access Object 
데이터베이스에 접속해 데이터의 추가, 삭제 수정 등의 작업을 하는 객체
코드의 유지보수 및 모듈화를 위해 별도의 DAO클래스를 작성하여 사용하는편

보통, 한 개의 테이블마다 한 개의 DAO클래스를 작성
DAO 클래스는 테이블로부터 데이터를 읽어와 자바 객체로 변환하거나 자바 객체의 값을 테이블에 저장.
DAO 클래스는 단 1개만 생성이 되도록, 싱글톤 패턴으로 만듭니다.
 > DAO 클래스는 단순히 DB연동만을 담당하므로 불필요하게 여러개의 객체르 만들필요가 없다. 따라서 싱글톤 패턴을 사용하여 1개만 생성하는것


VO - Value Object

VO에는 3가지 기본 특성이 있다.
1. Immutability(불변성) - 수정자가(setter) 없다
2. value equality(값 동등성) - 내부 값 동등성 검사
3. self validation(자가 유효성 검사) - 생성자에서 validate

DAO클래스를 구현하면 데이터베이스에 연결을 할 때, 데이터베이스 안에 있는 테이블의 속성을 가져오거나, 넣어주게 됩니다. ( select, insert, update )
그래서 항상, 데이터베이스 안에 있는 테이블의 컬럼과 같은 값을 갖는 자바빈 클래스를 만드는데, 이게 바로 VO클래스 입니다.
VO 클래스(Value Object) / DTO 클래스(Data Transfer Object) 라고 부릅니다.

DAO 클래스를 이용하여 데이터를 관리할때 데이터를 일반적인 변수에 할당하여 작업할수 있지만, 별도의 VO 클래스를 작성하여 데이터베이스와 관련한 변수들의 모음 역할을 합니다.
VO 클래스의 변수들은 외부에서 접근하지못하도록 private처리하고 getter, setter 메소드를 이용하여 조정

VO 클래스를 사용하는 이유는 데이터를 한곳에 관리할 수 있고 객체를 전달하기 쉬워 코드의 가독성이 증가하기때문 또한 보안상의 이유도있음
